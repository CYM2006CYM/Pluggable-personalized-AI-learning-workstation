# XH-202630 赛题要求对照表

> **同步说明**：学习者五维画像是学情可视化和建议层；CIDPP是教学内容质量评价，二者不得混用。D01已冻结，五维不驱动 `mastery`、路径或正式代码分数。

## 1. 事实来源

本表依据比赛 PDF。PDF 官方报名/提交日期是外部赛程事实；本项目实现使用六周相对周次，不将日期误写为内部开发截止。

## 2. 要求—实现—证据映射

| PDF 要求 | 产品实现 | 技术/数据证据 | 验收 |
|---|---|---|---|
| 垂直领域知识培训 | Pandas 数据清洗 Profile v2 | knowledge_index、source_map、datasets | 一个包端到端运行 |
| 不同背景学习者适配 | 计算机专业、非计算机自学、实践导向三类 persona | diagnostic、KnowledgeState、路径和卡片差异 | 每类至少三项可观察差异 |
| ≥3 个职责明确 Agent | 六角色串行协作 | 独立 AgentSession、结构化 artifact、trace | 生成—校验—决策闭环 |
| 动态个性化资源 | lesson、objective、code completion、practical task | Activity、模板、缓存和 Evidence | 至少三种资源可用 |
| 学情与路径可视化 | 画像状态、路径节点、原因、活动状态 | Web DTO、path snapshot | 演示可观察 |
| 讨论/辩论/交叉验证 | Hunter，实质争议时 Defender，Judge 裁决 | baseline/final、issues、decision | 20 组盲评资产 |
| 来源和数据合规 | source anchor、claim 溯源、合成数据、最小采集 | source_map、manifest、哈希、隐私说明 | 无未授权运行时数据 |
| 动态反馈 | ActivityEvidence 更新路径后缀 | evidenceVersion、changeReasons | 做题后下一步可解释 |

## 3. 评分维度对应

| PDF 维度 | 方案重点 | 交付证据 |
|---|---|---|
| 作品完整性 30 | 从诊断到活动、更新、恢复的闭环 | 三个完整案例、端到端视频 |
| 技术创新 25 | Profile 可插拔、确定性路径、串行对抗审核、双执行器适配 | 架构图、trace、接口文档 |
| 用户体验 15 | React 工作台、路径原因、加载/失败/恢复状态 | Web 截图和视频 |
| 实用价值 30 | Pandas真实任务、迁移题、量化指标 | 评测报告和原始数据 |

## 4. 硬指标口径

| 指标 | 分子 | 分母 | 目标 |
|---|---|---|---:|
| 幻觉率 | 判定为错误/杜撰的原子 claim | 全部已判定 claim | `<5%` |
| 难度匹配 | 命中 gold 可接受难度/脚手架集合的案例 | 可判定案例 | `≥85%` |
| 核心知识覆盖 | 满足 requiredEvidence 的核心知识点 | 核心知识点总数 | `≥90%` |

60 个案例、3 个差异化案例、20 组盲评和 6–10 人试测是团队内部强化验收资产，不声称为 PDF 的最低样本量。至少三组差异化案例按项目方案执行。

## 5. 提交物

设计文档、PPT、10 分钟内演示视频、源码/可执行程序、安装与系统说明、测试数据、专业知识包、差异化案例和效果报告。所有材料中的功能状态必须与本目录标签一致。

## 6. 主动说明的限制

- 多 Agent 是 SDK 能力范围内的串行隔离协作，不是并行群体网络。
- Node 评测服务面向本地受信任测试者，不是生产级恶意代码沙箱。
- 外部评测系统为调研/适配目标，失败时由 Node 方案兜底。
- Pandas 是唯一完整领域包；Profile v2 证明可扩展边界，不虚构第二领域。
- CIDPP用于教学内容优化，不作为学习者掌握度的正式标准；学习者五维画像已按D01进入上层展示，雷达图可降级隐藏但不能删除画像数据。GoA/OAEO实时链和复杂基础设施不属于当前MVP。
