# 02 可插拔资料包规范

> **D02/D03/D04 已冻结**：`profile.json`是 Profile v2 唯一清单；核心字段严格拒绝未知字段，允许 `x-` 扩展；`capabilities`声明的能力必须有对应资产。来源登记、官方事实裁决和 Pandas 七列清洗契约以 [第一周公共合同总册](./第一周任务/21-第一周公共合同总册.md) 为准。

> 版本：MVP-1.1；状态：六周实施规范。本文定义资料包如何描述“学什么”，不定义学习路径、代码执行器或前端页面的全部实现。

## 1. 目标和范围

资料包（Profile）是可安装、可校验、可启用、可修订和可归档的领域学习资产。应用只依赖统一契约，不把 Pandas、订单清洗或某个教材标题写死在代码中。

六周范围：本地资料包、一个完整 Pandas v2、一个极小非Pandas v2 smoke fixture、现有 `demo-review` v1 回归。不开放任意用户上传、远程抓取、运行时安装依赖、多租户后台或云数据库。

## 2. 当前事实与状态标签

| 标签 | 含义 |
|---|---|
| `[已有]` | 当前源码或 fixture 已实现并有回归测试。 |
| `[六周新增]` | 六周必须实现的 v2 能力。 |
| `[候选]` | 可在余力充足时实现，不影响固定主链。 |
| `[延期]` | 六周不实现。 |

当前 `pi-study-helper` 已支持 Profile v1：`profile.json`、`subject.md`、`knowledge_index.json`、`cards/`、`chapters/`、`exam_points/`、`source_map.json` 和 `quality_report.md`。`demo-review` 只作为 v1 加载、修订、启用、归档和旧学习流程回归样例，不是完整 v2 领域包。

Profile v2 的活动、诊断、数据、测试、Rubric、母题和环境能力均为 `[六周新增]`。普通单选、判断和简答的旧生成/判题基础 `[已有]`；Pandas v2 暂不把动态简答接入新主链。

## 3. 身份、版本和唯一清单

`profile.json` 是唯一清单文件，不新增第二个 `manifest.json`。它必须声明包身份、状态、版本和所有正式资产路径。

```json
{
  "subjectId": "pandas-cleaning",
  "name": "Pandas 数据清洗",
  "schemaVersion": 2,
  "status": "active",
  "version": "mvp-1.0.0",
  "revision": 1,
  "revisionOf": null,
  "capabilities": {
    "modalities": ["reading", "quiz", "code", "practice"],
    "runtimes": ["python-pyodide-v1", "python-node-v1"],
    "diagnostic": true
  },
  "paths": {
    "subject": "subject.md",
    "chapters": "chapters",
    "knowledge": "knowledge/knowledge-points.json",
    "goals": "goals/learning-goals.json",
    "cards": "cards",
    "activities": "activities",
    "diagnostic": "assessments/diagnostic",
    "assessments": "assessments",
    "rubrics": "rubrics",
    "datasets": "datasets",
    "taskGeneration": "task-generation",
    "referenceSolutions": "reference-solutions",
    "sources": "sources",
    "environments": "environments",
    "quality": "quality"
  }
}
```

- `subjectId` 是资料包族的稳定身份；不得因标题修改而改变。
- Profile v2不使用`slot`字段；`status`结合仓储所在的`active/draft/archived`区域表达生命周期。Profile v1原有`slot`继续只读兼容。
- `schemaVersion` 是目录和字段契约版本；`version` 是内容版本；`revision` 是同一资料包族的单调递增修订号。
- 正式资产任何变化都必须增加 `revision`：知识点、章节、来源、诊断、活动、数据、测试、Rubric、母题、注释、参考实现、环境引用和质量结论均包括在内。
- 只改受限构建日志不增加 `revision`。
- 会话创建时锁定 `subjectId + revision + schemaVersion`；旧会话继续使用旧修订，新会话读取 active。
- 资料包不保存 SDK 提交哈希；SDK 是项目内可调整组件。资料包只声明所需运行能力、活动契约和环境锁引用。
- 严格`ProfileManifestV2`、基础必填路径、`capabilities.diagnostic`和条件资产规则以21号公共合同D.2为唯一实现合同。本节JSON是符合该合同的Pandas实例，不再只是可自行增删字段的示例。

## 4. 目录结构和权限

```text
<profile-slot>/                              # 一个资料包修订根；Profile仓储读写，启用前不可对学习者开放
├─ profile.json                              # 唯一清单；服务端读取，记录身份、版本、能力和路径
├─ subject.md                                # 领域简介和目标；学习者可见，维护者编辑
├─ chapters/                                 # 章/节正文；学习者可见，按目录组织
│  └─ <chapter-id>/<section-id>.md           # 一个小节正文；通常对应一个主要知识点
├─ knowledge/                                # 知识点定义；PathEngine和诊断读取
│  └─ knowledge-points.json                  # 知识点、先修、来源、活动和关联编号
├─ goals/                                    # 正式学习目标；PathEngine读取，所有v2必备
│  └─ learning-goals.json                    # 目标知识点、必做活动和最终活动引用
├─ cards/                                    # 卡片、示例、反例和复盘；服务端裁剪后下发
├─ self-checks/                              # 无分自测；不产生正式掌握度证据
├─ activities/                               # 正式LearningActivity；服务端只读
│  └─ <activity-id>/                         # 单个活动稳定目录；可恢复、可审计
│     ├─ activity.json                       # 类型、知识点、题面、难度、来源和版本
│     ├─ starter.py                          # 代码活动起始代码；私有提交不可写回
│     └─ contract.json                       # 输入、输出和不变量契约
├─ assessments/                              # 诊断和正式评测索引；服务端受控读取
│  ├─ diagnostic/                            # 固定初始诊断；题面可下发，答案单独受限
│  │  ├─ questions.json                      # 8至12题的题面和知识点映射
│  │  └─ private/answer-key.json             # 正确答案、权重和阈值；不进前端或Agent
│  ├─ public/                                # 公开活动测试；可投影给执行器
│  └─ private/                               # 学习活动私有测试；不属于05号正式保留集
├─ rubrics/                                  # 评分维度、权重、阻断项和通过线
├─ datasets/                                 # 固定公开/私有数据；不含用户答案
│  ├─ fixtures.json                          # fixture稳定ID、可见性、CSV路径和SHA-256统一索引
│  ├─ public/                                # 公开学习数据；前端按安全包使用
│  ├─ private/                               # 服务端正式评测数据；不下发
│  └─ tooling/                               # 维护和复现脚本；不进入运行主链
├─ task-generation/                          # 母题和构建期生成约束
│  ├─ mother-tasks/                          # 少量人工编写母题；维护者确认
│  ├─ thinking-atoms/                        # AI抽取、人工确认的可审核原子
│  ├─ variant-plans/                         # 确定性组合计划；构建阶段使用
│  └─ accepted-variants/                     # 已审核固定变式索引；服务端只读
├─ reference-solutions/                      # 参考实现；仅构建校验和评测器读取
├─ sources/                                  # 来源元数据、摘录和锚点
├─ environments/                             # 运行能力声明
│  └─ environment-lock.json                  # 原型验证后冻结的版本与限制
└─ quality/                                  # 质量报告和启用摘要
   ├─ quality-report.md                      # 人工可读报告
   └─ quality-report.json                    # 机器可读问题和门禁结果
```

`datasets/fixtures.json`是测试数据引用的唯一索引，顶层固定为`{"fixtures": [...]}`。每项只允许`fixtureId/visibility/fileRef/format/assetHash`五个字段；Pandas六周包只允许CSV。活动的`datasetRefs`和测试的`fixtureRefs`均填写稳定fixture ID，不直接填写文件路径。公开测试只能引用公开fixture，隐藏测试可引用公开或私有fixture；完整闭合和错误归因以21号D.6为权威。

`_user/` 学习者私有数据不属于资料包目录，不随资料包分发。05号正式效果评测的 `runtime/hidden/`、人工标准和保留答案也不放入资料包。

## 5. 章节、小节和知识点

MVP 目录固定为“章→节→知识点”三级；更深标题留在小节正文中。知识点数量由学习材料决定，不写死为六个。

```ts
interface KnowledgePointDefinition {
  id: string;                       // 稳定ASCII编号；标题修改不得复用或改变
  title: string;                    // 学习者可见名称
  chapterId: string;                // 主要所属章节
  sectionId: string;                // 主要所属小节；MVP一节建立一个主要知识点
  prerequisiteIds: string[];        // 先修知识点；必须无环且均存在
  relatedKnowledgePointIds: string[];// 相似/关联点；只提示关系，不自动合并掌握度
  sourceAnchorIds: string[];        // 专业事实来源锚点
  activityIds: string[];            // 可验证活动引用
  importance: number;               // 0..1重要度；由维护者确认
}

interface LearningGoalsAsset {
  goals: LearningGoalDefinition[];    // 顶层命名容器；禁止裸数组
}

interface KnowledgePointsAsset {
  knowledgePoints: KnowledgePointDefinition[]; // 知识点ID固定读取id
}

interface LearningActivitiesAsset {
  activities: LearningActivity[];     // 活动ID固定读取activityId
}
```

三类JSON容器、ID字段、重复检测、引用存在性和先修环校验以21号公共合同D.2.1—D.2.3为唯一实现规范。目标文件必须使用`{"goals": [...]}`，知识点文件必须使用`{"knowledgePoints": [...]}`，活动文件必须使用`{"activities": [...]}`；不得使用裸数组或自定义`items`容器。这里的`LearningGoalDefinition`和`LearningActivity`分别引用21号D.2与12号正式活动联合类型，不在02号另建第二套字段。

同一概念出现在多个小节时，每节仍建立自己的知识点；使用 `relatedKnowledgePointIds` 记录关联，不能按标题自动合并证据或路径节点。一个知识点必须能够独立讲解、独立识别至少一种错误，并有至少一种诊断或学习验证活动。

构建流程为：AI抽取目录和知识点草稿 → 程序校验引用、编号、先修环和重复 → 维护者确认或合并/拆分 → 生成活动和来源影响报告。无目录材料也只能生成 draft，不能自动启用。

## 6. 来源、断言和许可证

官方文档、教材和公开教程均可参与内容设计；发生技术冲突时以官方文档为裁决依据，并在质量报告记录原因。来源不要求整套复制：许可证允许时可保存较完整内容，否则保存必要摘录和元数据。

```ts
interface SourceAnchor {
  sourceAnchorId: string;           // 稳定来源锚点编号
  kind: "official" | "textbook" | "tutorial"; // 来源层级
  title: string;                   // 来源名称
  locator: string;                 // 页面、章节或文件定位
  version?: string;                // 采用版本或访问日期
  excerpt?: string;                // 允许保存的必要摘录
  excerptHash: string;             // 摘录摘要，检测内容变化
  license: string;                 // 许可证或再分发说明
}
```

每个可验证专业断言、题面事实和关键讲义结论必须绑定一个或多个来源锚点。定位失效、摘要不匹配、许可证缺失或来源冲突未裁决时，标记 `unverifiable`，禁止进入正式活动或使用固定模板回退。

## 7. 活动、诊断和客观题

02号的资料包能力模态与12号运行活动分层：

| 资料包模态 | 运行时可能映射 | 说明 |
|---|---|---|
| `reading` | 卡片、章节、`explain` | 教学材料，不必产生正式计分活动 |
| `quiz` | `mcq` | 单选/判断正式题；无分自测不计分 |
| `code` | `code_completion`、`debug` | 代码活动 |
| `practice` | `coding_practical` | 综合实操 |

### 7.1 初始诊断

- 资料包固定8至12道单选/判断，10分钟以内；题面和答案分离。
- 六个核心点（若资料包有不同数量则覆盖全部核心目标）均需有直接证据，另可抽样最小前置能力。
- 学习者主动请求或相关资产修订时创建新诊断版本，不覆盖旧快照。

### 7.2 学习后普通题

学习后的单选/判断可以沿用当前 `study_generate_question` 能力动态生成：

```text
AI生成单选/判断
→ 程序校验题型、选项、答案、知识点、难度和来源
→ 单一CIDPP评价Agent做一次轻量质量评价
→ 仅高风险事实争议进入Hunter/Judge
→ 展示题目
→ 代码比较用户答案与correct_answer
→ 保存Evidence
```

AI只能改变难度、情境、典型错误针对性和措辞，不能改变知识边界、来源事实、正确答案标准或先修关系。Pandas v2暂不把动态简答接入新主链；旧v1简答流程继续兼容。

每个知识点的学习后题组由AI决定4至6题及固定/动态比例，优先动态题。固定题不是主要题库：每个核心知识点默认仅准备1至2道固定兜底题，动态题不足时使用未做过的固定兜底题；固定题也不足以组成4题时结束本轮并标记“验证证据不足”，不阻塞、不虚报通过。通过线为75%，最低答对数为 `ceil(questionCount × 0.75)`。每题保存明细证据；同一题组在置信度和证据形式数量中只计一次相关验证。

固定客观题只承担三种用途：固定初始诊断、动态生成故障兜底、05号正式评测。三类资产必须分目录、分引用、分权限，不能互相混用。

## 8. 代码任务资产边界

Pandas 六周代码补全和独立实操均使用 `profile_fixed`：题面、起始代码、可编辑区、固定数据、公开/私有测试、参考实现、Rubric和环境锁随资料包交付。每个核心代码知识点至少一个固定兜底活动，代码补全母题共4个，最终实操母题1个。

完整 `ai_generated` 代码题方案保留为12号候选扩展，不是Pandas六周主链。母题和思维原子在轻量方案中用于构建期生成和审核三级中文注释、题面表达、难度与错误针对性；不在学习时生成新的测试或参考实现。

## 9. CIDPP内容质量评价

CIDPP只评价教学内容质量，不评价学习者掌握度。五个维度为：清晰性、完整性、深度、实用性、针对性；采用0至100分，作为优化依据和人工参考，不因单项差1分自动阻断。

构建期：程序事实门禁 → 单一CIDPP评价Agent → 最多一次优化 → 高风险才Hunter/Judge → 核心资产人工确认。运行时动态普通题只做一次轻量评价，超时或失败使用固定题。质量报告保存最终摘要，完整中间产物放受限构建记录。

## 10. 构建、启用、修订和恢复

所有v2资料包必须提供`subject/chapters/knowledge/goals/sources/quality`六项基础路径；声明的能力再触发21号D.2规定的条件资产。Pandas正式包声明`diagnostic=true`，因此还必须提供诊断题面、私有答案和知识点映射。未声明诊断的smoke资料包可以省略诊断资产，但不得伪造诊断完成状态。

`/study-build` 生成 draft 骨架、知识点、讲义、来源候选和活动草稿；维护者补齐/确认业务契约、诊断答案、母题、参考实现、私有测试和Rubric。启用前必须通过结构、引用、内容、安全、可运行和人工核心资产门禁。

启用流程：draft 完整写入临时目录 → 解析和交叉引用校验 → 测试/参考实现预检 → 质量报告 → 比较 active revision → 原子提升为 active。失败保留 draft，旧 active 不变。启用冲突返回 `revision_conflict`，要求基于最新 active 重新修订。

## 11. 质量门禁和验收

- v1 `demo-review` 回归不破坏；smoke fixture 只验证v2加载和最小活动，不出现在正式产品选择页。
- 所有路径安全、文件存在、版本一致；目标、知识点、活动按21号D.2固定顺序完成重复、悬空、主辅边界和先修无环校验。
- 每个正式活动均有来源、知识点、契约、判定方式和失败回退。
- 诊断答案、活动私有测试、参考实现和05号保留评测资产不进前端、普通日志、Agent输入或质量摘要。
- Pandas固定数据和参考实现可在锁定环境重复运行；具体版本和资源阈值只能来自原型冻结。
- 正式效果指标如幻觉率、难度匹配、学习增益和路径合法率由05号统一评测；02号只报告资料包资产完整性和可运行性。

## 12. 已冻结、候选和延期决策表

| 项目 | 六周状态 | 结论 |
|---|---|---|
| Pandas代码补全/实操来源 | `[六周新增]` | 固定 `profile_fixed` |
| 普通单选/判断 | `[已有+六周修正]` | AI动态生成优先，代码比较答案，失败用少量兜底 |
| 动态简答 | `[延期]` | 不进入Pandas v2新主链 |
| CIDPP | `[六周新增]` | 单一评价Agent，内容质量优化，不替代掌握度 |
| 学习者编程五维 | `[六周新增][已冻结]` | 方案B正式上层画像；AI直接评分可观察维度，确定性底座仍由程序维护 |
| 完整AI代码题生成 | `[候选]` | 12号保留技术设计，六周不进入Pandas正式主链 |
| 任意资料包上传后台 | `[延期]` | 只加载内置/维护者目录 |
