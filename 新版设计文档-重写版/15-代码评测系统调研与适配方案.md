# 代码评测系统调研与适配方案

> **D07同步**：Node本地执行器是六周正式提交权威；Pyodide只负责公开预览。执行协议、双进程、结构化 `evaluator-result.json`、DataFrame容差 `atol=1e-6`/`rtol=1e-5`和错误优先级以 [第一周公共合同总册](./第一周任务/21-第一周公共合同总册.md) 为准。具体版本和资源上限必须由原型写入环境锁。

## 1. 定位

成熟评测系统是代码执行/判分的实现选项，不是新增产品需求。六周必须有可用的 Node 本地评测兜底；外部系统列为 `[调研]`，不因其接入失败阻塞演示。

## 2. 统一适配边界

代码评测端口的唯一接口规范位于12号文档“服务端 `CodeEvaluationPort` 与浏览器运行接口”一节。本文件不再维护第二套方法签名、输入类型或结果类型，候选评测器必须实现12号规定的：

- `prepare(input: PrepareEvaluationInput): Promise<PreparedEvaluation>`：校验活动、资料包修订、环境锁和任务资产摘要，并准备服务端私有评测状态；
- `run(input: RunEvaluationInput, signal: AbortSignal): Promise<ActivityResult>`：在取消/超时信号约束下执行一次预览或正式提交，并返回统一结果。

其中 `PrepareEvaluationInput`、`PreparedEvaluation`、`RunEvaluationInput`、`EnvironmentLock`、`ActivityResult`、`EvaluationMode` 和错误枚举均以12号文档为准；本文件中的“任务编号、环境版本、timeout”等说明只能理解为这些字段的业务含义，不能重新定义字段结构。候选适配器不得暴露参考答案、隐藏断言、宿主文件、接口密钥或网络凭据。

六周实现顺序固定为：Node.js/Python本地适配器先满足12号正式提交语义；Pyodide只实现公开运行对应的 `BrowserCodeRunner`；外部评测器若通过本文件第6节门禁，只替换服务端端口实现，不改变活动、HTTP、Attempt、Evidence、Rubric或页面契约。

## 3. 候选比较维度

调研候选（例如 Judge0 类服务、在线判题框架、容器化 runner）时按 Python/Pandas 支持、隐藏测试、超时/内存限制、离线安装、API 复杂度、许可证、Windows 本地可运行性、数据隔离和六周适配工时打分。每个候选须完成一个补全和一个 Pandas 实操原型，而非只看 README。

## 4. Node 兜底实现

Node 服务启动受控 Python 子进程，传入只读数据副本和已准备的任务资产引用；限制环境变量、cwd、stdout/stderr、墙钟超时、临时目录和允许库。正式提交在全新进程中运行公开后隐藏测试，测试异常记 `evaluator_error`。本方案面向本地受信任测试者，不宣称生产级恶意代码沙箱；Docker/系统沙箱是赛后升级。所有具体版本、限制和能力标志必须写入12号定义的 `EnvironmentLock`，不能在本文件另造 `EnvironmentSpec`。

## 5. Pyodide 双后端一致性

浏览器 Worker 用锁定 Pyodide/Pandas 做即时预览和公开测试；Node 做正式提交。第 5 周用同一任务和 10 组输入比较通过/失败及关键输出；差异必须归因于环境版本并记录，不得静默覆盖。若外部平台或 Pyodide 不稳定，Node 结果作为最终演示判分，Pyodide 只显示预览不可用。

## 6. 调研门禁与退出

W1 依据12号接口定义端口和安全需求；W4 完成候选原型、许可证和安装成本记录；W5 以离线重跑、异常分类、隐藏资产隔离和 Windows 安装成功为门禁。任何候选不满足四项即退出并采用Node；不为调研引入FastAPI、消息队列或云部署。候选矩阵记录的是适配器实现证据，不得成为第二套公共接口或第二套资料包目录规范。
