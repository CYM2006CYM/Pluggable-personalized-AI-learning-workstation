## 3. 统一状态标签

- **[已有]**：源码和测试已经证明，不能仅凭设计文档认定。
- **[MVP]**：六周必须实现并有自动验收。
- **[演示]**：最终展示必须可观察，通常依赖 MVP。
- **[可降级]**：时间不足时使用的可运行替代。
- **[候选]**：可能采用但尚未锁定的方案。
- **[调研]**：必须输出比较、原型或结论后才能决定。
- **[待决策]**：仍需负责人确认，不能作为实现事实。
- **[延期]**：六周明确不做，仅保留未来接口或说明。

## 4. 全局术语和一致性规则

| 术语 | 统一定义 |
|---|---|
| Profile | 可插拔领域资料包；`profile.json` 是唯一身份入口 |
| Profile v1 | 当前复习包格式，含 subject、cards、chapters、exam_points、knowledge_index、source_map、quality_report |
| Profile v2 | 在 v1 上增加 goals、activities、assessments、rubrics、datasets、capabilities 和知识点级溯源 |
| active/draft/archived | active、draft 是 manifest/slot 状态；archived 是旧 active 的目录生命周期，不写成第三种当前 status |
| LearningActivity | lesson、self_check、objective_question、code_completion、practical_task 五类活动联合类型 |
| Evidence | 可追溯的学习行为证据；Agent 解释不是事实证据 |
| KnowledgeState | 知识点 mastery、confidence、status、错误标签和证据引用 |
| PathEngine | 由确定性代码生成合法路径；Agent 只能解释路径理由 |
| baseline/final | 同一生成原稿在审核前后的两个版本，用于公平单/多 Agent 对比 |
| evaluator_error | 评测器故障，不得计为学习者答错或降低 mastery |

全局约束：

1. `profileRevision`、`sessionId`、`generationRunId`、`evidenceVersion` 是跨模块追踪字段。
2. 固定知识、来源、数据集、Rubric、公开/隐藏测试由维护者控制；Agent 不能改写。
3. 章节学习和个性化路径共用同一 PathEngine、Session、Evidence 和 Profile，不建设第二套路由系统。
4. 章节入口可以是 [已有]，目标/路径向导是 [MVP]；不能把两者写成两个互不相干的产品。
5. React 前端不直接访问 Profile 文件、仓储、模型 SDK 或隐藏测试。
6. 赛题硬指标是离线演示评测目标，不是生产线上 SLO。

## 5. P0 跨文档闭环检查

P0 功能不是在每篇文档重复全文，而是必须在相关文档分别回答自己的问题，并共享同一字段、状态和错误语义。最低闭环为：

| 功能 | 产品入口 | 应用用例 | 核心实现 | 数据记录 | 测试 | 计划 |
|---|---|---|---|---|---|---|
| 初始诊断 | 诊断向导 | `submitDiagnosticAnswer`（`submitDiagnostic`为兼容别名） | 固定题 + EvidenceAggregator + Diagnosis Graph | LearnerDiagnostic、Evidence | 诊断计分、矛盾证据、恢复 | W2 |
| 学习路径 | 路径确认页 | `buildPath` | PathEngine + Decision Graph | LearningPath、KnowledgeState | 先修、预算、难度、100%合法 | W2-W3 |
| 学习卡片 | 学习工作台 | `openPathNode` | Generate → Hunter → Judge | Artifact、source refs、trace | claim、来源、fallback、泄漏 | W3-W4 |
| 代码补全 | 代码工作台 | `submitActivity` | Node正式评测 + CodeEvaluationPort；W5增加Pyodide预览 | ActivityAttempt、ActivityEvidence | 正确、错误、超时、评测器故障 | W3-W5 |
| 独立实操 | 任务页 | `submitActivity` | Node隐藏测试 + Rubric；W5增加双后端对照 | score、dimensionResults、Evidence | 答案隔离、重复运行一致 | W3-W5 |
| 动态更新 | 下一步/总结 | `advancePath` | EvidenceAggregator + PathEngine | state/path version | 变化理由、已完成节点不重排 | W3-W5 |
| 多 Agent 审核 | 资源审核详情 | `generateArtifact` | 串行隔离 Graph | baseline/final、issues、checkpoint | 条件 Defender、trace、fallback | W4 |
| 会话恢复 | 恢复页 | `recoverSession` | checkpoint repository | sessionVersion、checkpoint | 刷新、超时、磁盘失败、CAS | W3-W5 |

## 6. 变更治理

如果需要改变公共字段、状态值、错误类型或验收指标，必须同时更新：

1. 产生该契约的专项文档；
2. 01 总体设计；
3. 05 测试口径；
4. 06 周计划和门禁；
5. Pandas fixture 或示例；
6. 17 审计记录。

任何“当前已有”声明都必须能指向源码路径、测试文件或可重现命令。
