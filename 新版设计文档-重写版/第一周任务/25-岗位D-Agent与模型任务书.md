- 19号：只阅读 Agent 工作流、交接、fixture和越权规则。
- 20号：只阅读D01、D06、D10、D13和D14；这些决定画像、串行编排和候选边界。
- 21号：完整阅读合同A的画像权限、合同C的模型与角色合同；合同B只阅读上下文提问和Facade调用边界。
- 03号：阅读当前九个 Graph、隔离执行器和串行协作事实。
- 11号：阅读确定性掌握度、路径和五维画像的不可越权边界。
- 12号：阅读活动任务来源和 Agent 不得生成正式代码题的六周边界。
- 18号：阅读提示版本、交接、变更和跨岗位审计规则。

### 不作为本周设计依据

- 02、07号具体来源和 Pandas 内容；D 不决定专业事实。
- 04号页面展示；D 不决定前端文字、布局或交互。
- 05、06号正式指标；D 不修改人工标准、路径合法率或评测门槛。
- 14号完整AI代码题候选方案；D 只能记录未来接口约束，不能把它接入P0。

如果当前 Graph 的真实能力与规划文档不同，以源码和测试事实为准，并上报差异；不得为了匹配设计文档伪造已有能力。

## 执行边界

### 本周允许执行

- 清点现有 Graph 的输入、输出、调用方、取消和失败行为。
- 实现 `ModelExecutionPort` 适配、录制响应、角色 schema 校验和 `ReviewOrchestrator` 串行骨架。
- 制作 Generator、Hunter、条件Defender、Judge、CIDPP和CapabilityScorer的正常/失败/超时 fixture。
- 实现安全上下文裁剪、promptVersion/modelId记录、checkpoint和固定 fallback。
- 对五维画像只保存 AI 返回的可观察维度结果及其 Evidence 引用，不重算掌握度。

### 本周禁止执行

- 不得让 Agent 决定客观答案、代码分数、Rubric阻断、Evidence、KnowledgeState、路径或先修关系。
- 不得向 Agent 发送隐藏测试、参考实现、完整Rubric、诊断答案、密钥或宿主路径。
- 不得把 Agent 的解释写成正式掌握度证据，不得让 CIDPP 分数进入学习者画像。
- 不得实现并行 Agent 网络、无限自我修订、实时自动路径决策或完整AI代码题主链。
- 不得固定 SDK 提交哈希或把当前模型版本写成永恒依赖。

### 缺口处理

发现角色输入或输出未定义时，只能做录制 fixture 和问题单；不得自行扩展公共 schema。发现模型失败时，必须验证 fallback，不能用“模型这次回答合理”代替确定性校验。

## 任务

1. 清点当前九个 Graph 的输入、输出、调用方和已有能力。
2. 按 21 号合同实现 `ModelExecutionPort` 适配边界。
3. 为 Diagnosis、Generator、Hunter、Defender、Judge、CIDPP 和解释角色准备输入/输出 fixture。
4. 实现 Generator→Hunter→条件Defender→Judge 的严格串行、有限重试、checkpoint 和 fallback 适配。
5. 建立安全输入过滤：不得把诊断答案、隐藏测试、参考实现、完整 Rubric、密钥或宿主路径传入 Agent。
6. 准备录制响应/固定响应适配器，记录 promptVersion、modelId 和失败原因。

## 交付物

- 当前 Graph 清点记录；
- 模型端口适配和角色 fixture；
- Agent 权限/禁止输入矩阵；
- 串行编排和失败降级记录；
- `handoff-w1-d-*` 交接记录。

## 不负责

不修改客观答案、代码分数、Rubric 阻断、Evidence、KnowledgeState、路径、先修关系和隐藏测试；不实现完整 AI 代码题生成。

## 验收

A 检查 Agent 不能写权威事实，B 检查来源和专业边界，C 检查不改变评测结果，E 扫描 fixture 不泄漏隐藏资产。
