## 阅读边界

### 必须阅读

- 19号：只阅读资产制作、交付、隔离和问题单规则。
- 20号：只阅读D02、D03、D04、D10和D12；这些决定来源、数据、Profile和任务来源。
- 21号：完整阅读合同D；合同A只阅读Evidence、revision和知识点引用，合同E只阅读Rubric、测试资产和隔离部分。
- 02号：阅读Profile v2目录、清单、来源、许可证、质量门禁和revision规则。
- 07号：阅读Pandas章节、六个知识点、固定数据、固定任务和 `clean_df` 目标。
- 11号：阅读诊断资产如何关联知识点和Evidence。
- 12号：阅读固定代码任务、公开/隐藏测试和Rubric边界。

### 不作为本周设计依据

- 03号 Agent 角色细节；B只提供可引用来源和安全内容，不设计提示词。
- 04号网页呈现；B只提供页面需要的题面、反馈和安全摘要。
- 09、10号源码适配；B不改变源码目录和应用接口。
- 13号五维能力评分；B只标记活动可观察维度，不决定评分算法。
- 14号完整AI代码题生成；六周固定任务不按候选方案制作运行服务。

未完成的讲义、来源或题目要求只能形成候选资产；不能因为模板已经完成就标记为 `active`。

## 执行边界

### 本周允许执行

- 制作来源登记、许可证、定位、摘要哈希和冲突记录模板。
- 制作 Profile 资产目录、章节/知识点/诊断/固定任务/Rubric 模板。
- 制作候选 CSV、参考清洗脚本、异常分布和 `clean_df` 验证案例。
- 为每项资产标记 `draft`、`active` 或 `evaluation-only` 的建议状态，但最终启用由负责人批准。
- 记录每个资产的生产者、读取者、私有性、恢复能力和revision影响。

### 本周禁止执行

- 不得自行选择最终教材、教程、许可证、城市映射扩展或未批准来源。
- 不得改变七列顺序、日期/金额/状态清洗、重复选择或 `clean_df` 契约。
- 不得把诊断题、学习兜底题和正式评测题混放。
- 不得把隐藏测试、参考实现、完整Rubric或诊断答案放进公开资产。
- 不得制作完整AI代码题生成、测试生成或参考实现生成服务。

### 缺口处理

遇到专业事实冲突，记录双方来源和定位，标记 `pending_owner_decision`，不得自行用“常识”裁决。遇到Schema无法承载资产，提交字段缺口，不在资料包中增加未批准字段。

## 任务

1. 按 21 号合同整理 Profile v2 资产目录、`profile.json.paths`、读写者、私有性和 revision 影响。
2. 制作来源登记模板和冲突记录模板；不自行裁决官方事实冲突。
3. 制作知识点、章节、讲义、固定诊断、固定代码任务和 Rubric 模板。
4. 按 D12 记录诊断题、学习兜底题、正式效果评测题的隔离路径。
5. 制作 Pandas 来源候选清单、七列数据候选、异常分布和 `clean_df` 验证计划。
6. 准备第二周 Pandas 资料包资产清单、缺口表和人工确认表。

## 交付物

- 来源登记模板和候选列表；
- Profile 资产目录表；
- 内容、诊断、固定任务和 Rubric 模板；
- Pandas CSV 候选及清洗测试草案；
- `handoff-w1-b-*` 交接记录。

## 不负责

不修改 Facade、执行器协议、路径算法、Evidence 权重、错误码或模型端口；不把候选 CSV 当成 active 正式资产，必须等负责人批准。

## 验收

A 检查资产能落入 Profile schema，C 检查任务能被评测，E 检查来源和人工标准记录，负责人批准后才能进入第二周正式制作。
