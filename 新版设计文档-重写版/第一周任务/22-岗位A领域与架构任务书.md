## 阅读边界

### 必须阅读

- 19号：只阅读第一周目标、权限规则、交付和验收部分。
- 20号：只阅读D04、D05、D08、D11和D14；这些是本岗位的冻结决策。
- 21号：完整阅读合同A和合同B；合同C—E只阅读与类型、Facade、Evidence和错误码交接有关的章节。
- 09号：阅读当前分层、已有仓储、命令和Graph事实。
- 10号：阅读v1兼容和新增适配边界。
- 11号：阅读KnowledgeState、Evidence、路径版本和恢复规则。
- 16号：阅读统一入口的调用场景，不把其中旧接口名称当成新接口。

### 不作为本周设计依据

- 04号网页视觉和交互细节；A只提供安全DTO需要的领域字段。
- 07号具体 Pandas 内容和数据值；A只校验它们能否符合 Profile schema。
- 12号执行器内部实现；A只消费 `ActivityResult` 合同，不重写评测语义。
- 13号五维画像展示细节；A只保存版本和证据引用，不设计新的画像算法。

如果上述文档存在未完成段落，A不得补写产品规则；只记录“文档位置—缺口—受影响接口—建议负责人决定”。

## 执行边界

### 本周允许执行

- 只在 `src/domain/`、`src/application/`、`src/repositories/` 和合同指定的测试目录实现骨架。
- 只实现21号合同已经写明的字段、枚举、方法、版本、错误码和事务边界。
- 可以建立接口、解析器、校验器、适配别名、非法 fixture 和回归测试。
- 可以为尚未实现的业务方法返回明确的未实现结果，但不能伪造成功事实。

### 本周禁止执行

- 不得新增第二个 Facade、第二套状态、第二套错误码或第二套 Profile manifest。
- 不得改变 `mastery` 公式、路径排序、先修关系、Rubric、Evidence权重或五维评分算法。
- 不得把旧 `LearningProfile` 或总正确率转换成知识点掌握度。
- 不得实现网页、实时模型调用、完整 PathEngine 或完整代码执行器来替代其他岗位。
- 不得为了让类型通过而放宽核心字段、路径安全或未知关键字段拒绝规则。

### 缺口处理

发现公共合同不够实现时，停止扩展该接口，提交问题单；负责人批准后才能修改21号总册和本任务的实现。

## 任务

1. 清点当前 Profile v1、五个 Pi 命令、StudySession、Attempt、SessionEvidence、LearningProfile、仓储和 Graph。
2. 按 21 号合同落地 `src/domain/v2-types.ts`，不新增公共字段或状态。
3. 按 21 号合同实现 Profile v2 解析骨架、路径安全、坏包拒绝和 v1 只读兼容。
4. 按 Facade 17 方法建立 `src/application/learning-runtime-facade.ts` 骨架；旧名称只做适配别名。
5. 建立版本、幂等、错误码、安全 DTO 和恢复检查点的最小测试。
6. 建立 `LearningSessionRepository` 的接口骨架，但不越过 D11 直接由活动仓储提交 Evidence。

## 交付物

- 公共类型和 Facade 实现或明确失败记录；
- 至少一个非法 Profile fixture；
- v1 回归记录；
- 字段、状态、版本和错误码映射表；
- `handoff-w1-a-*` 交接记录。

## 不负责

不制作 Pandas 专业内容，不决定来源、数据清洗规则、路径评分权重、代码 Rubric 或模型提示词；发现冲突提交问题单。

## 验收

E 检查安全 DTO 和坏包测试，B 检查 Profile 字段语义，C 检查 ActivityResult 接入，D 检查 Agent 权限；旧流程不能回归失败。
