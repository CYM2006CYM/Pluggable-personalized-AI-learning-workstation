# 任务生成、缓存与评测资产策略

> **D10/D12 已冻结**：六周 Pandas 代码任务固定使用 `profile_fixed`；完整 AI 代码题仅为候选。学习测试和正式评测隐藏集使用固定物理路径，具体边界见 [20号冻结记录](./第一周任务/20-第一周开发前负责人决策冻结清单.md) 与 [第一周公共合同总册](./第一周任务/21-第一周公共合同总册.md)。

> **D10/D12同步**：六周固定代码任务使用 `profile_fixed`；完整AI代码题生成不创建P0接口、缓存或运行服务。学习活动私有测试固定位于 `profiles/.../activities/.../tests/private/`，正式评测隐藏集固定位于 `evaluation/runtime/hidden/`，不再由实现决定是否物理分离。

## 1. 原则

任务不是让 Agent 从空白发明。Profile 固定知识点、目标、任务模板、数据 schema、参考实现、公开/隐藏测试和 Rubric；Agent 只填写题面、示例、脚手架和提示的允许字段。质量和成本可预测优先于无限变体。

## 2. 生成层级

- `[MVP]` 预生成并审核每个核心节点的 fallback lesson、客观题、补全和实操。
- `[MVP]` 按需生成仅用于非关键措辞和画像适配，失败立即回退。
- `[候选]` 批量离线生成可用于资料包修订，不进入实时主链。

## 3. 缓存键与失效

```text
cacheKey = hash(profileId, profileRevision, knowledgePointId, activityKind,
                learnerBand, scaffoldLevel, taskTemplateVersion,
                promptVersion, modelId, locale)
```

缓存值包含 artifact、claim refs、审核状态、generationRunId、token、耗时和创建时间。Profile revision、模板、提示、模型、来源哈希或测试资产任一变化即失效；不覆盖旧终稿，保留可审计链。缓存只存脱敏内容，禁止 API key、隐藏断言和参考答案进入前端缓存。

## 4. 质量门与成本预算

生成后依次做 schema/白名单校验、来源 claim 校验、代码预运行、Hunter；高风险（代码、版本敏感、来源不足）才触发 Defender；Judge 决定 publish/fallback/revise。记录每次调用 token、延迟、重试次数和缓存命中率。MVP 每个核心节点最多 2 次 revise，实时等待超过 15 秒回 fallback；第 4 周根据实测调整预算但必须写入报告。

## 5. 评测资产隔离

公开测试元数据可供学习者；隐藏测试、reference solution、Rubric blocking 规则仅由 Node/外部评测器读取。学习活动私有测试固定放在 `profiles/.../activities/.../tests/private/`，正式效果评测隐藏集固定放在 `evaluation/runtime/hidden/`；任务生成器收到的 ContextView 不含隐藏资产。

## 6. 一致性测试

相同 key 必须命中同一 artifact；不同 revision 不得命中旧缓存；缓存删除后可从模板和固定 seed 重建；fallback 在模型不可用、Judge 失败和网络断开时可用。报告命中率、平均延迟、p95 延迟、每任务 token 和失败原因。
